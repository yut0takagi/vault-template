# vault-template

Obsidian × Claude Code のテンプレート。  
clone して開くだけで、ナレッジ管理・議事録自動生成・AI自動化が揃った Vault が手に入る。

## クイックスタート

```bash
git clone https://github.com/yut0takagi/vault-template.git my-vault
cd my-vault
```

1. **Obsidian** で `my-vault` を Vault として開く
2. **Claude Code** で `/初期設定` を実行 → MCP・プラグインを対話的にセットアップ
3. 使い始める

## できること

### 情報を集める

| スキル | やること |
|--------|---------|
| `/リサーチ` | Web検索→ナレッジに自動保存 |
| `/Webクロール` | 複数ページを再帰的に取得 |
| `/Web取り込み` | 単一URLをナレッジ化 |
| `/defuddle` | Webページからクリーンなmarkdown抽出 |

### 議事録・ナレッジを作る

| スキル | やること |
|--------|---------|
| `/議事録作成` | 音声/文字起こし→NotebookLMで文字起こし→議事録md生成（並列処理・Whisperフォールバック対応） |
| `/obsidian-markdown` | Obsidian記法のmarkdown生成 |
| `/obsidian-bases` | Obsidian Bases（DB機能）でデータベース作成 |
| `/json-canvas` | JSON Canvasでマインドマップ・関係図 |

### 成果物を作る

| スキル | やること |
|--------|---------|
| `/リサーチ2スライド` | リサーチ→ナレッジ→スライドまで一気通貫 |
| `/スプレッドシート` | Excel(.xlsx)作成・編集・分析 |
| `/ドキュメント作成` | Word(.docx)生成 |
| `/PDF操作` | PDF結合・分割・テキスト抽出 |
| `/GASコード作成` | Google Apps Script生成 |

### テスト・検証

| スキル | やること |
|--------|---------|
| `/Webテスト` | PlaywrightでWebサイトの動作テスト・スクリーンショット |
| `/深掘り` | アイデアや計画を容赦なく質問して精緻化 |

### 文章を磨く

| スキル | やること |
|--------|---------|
| `/人間味` | AI臭い文章を人間らしく書き直す |

### セットアップ

| スキル | やること |
|--------|---------|
| `/初期設定` | MCP・プラグイン・Obsidianを対話的にセットアップ |

## フォルダ構成

```
inbox/                     未整理素材の入口
  raw/                       生データ
  temp/                      一時ファイル（git管理外）
  まとめ済み/                 処理済みインプット
  ストック/                   保留・あとで読む

ナレッジ/                   知識ベース
  技術/ ツール/ リサーチ/
  README.md                  Map of Content（インデックス）

議事録/                     会議・面談記録
日報/                       日報・振り返り
タスク/詳細/                タスク個票
制作/スライド/              発表資料
ログ/チャット/              AIチャットログ

添付/                       画像等バイナリ（Obsidian添付先）
アーカイブ/                 処理済みファイル退避
Excalidraw/                 図・ホワイトボード
Clippings/                  Webクリップ

設計/                       Vault設計ドキュメント
  ARCHITECTURE.md             設計思想書
  タグ定義.md                  タグ運用ルール
  テンプレート/                日報・月報テンプレート

.claude/                    Claude Code自動化
  skills/ (18個)              スキル
  agents/                     エージェント定義
  commands/                   カスタムコマンド
  settings.json               権限設定

.obsidian/                  Obsidian設定
CLAUDE.md                   プロジェクトルール
```

## 設計思想

| 原則 | 意味 |
|------|------|
| テキストは中に、バイナリは外に | mdはVault内、音声・動画は外部ストレージ |
| フォルダは役割、タグは性質 | フォルダ=「何か」、タグ=「何について」 |
| 1ファイル1目的 | 議事録とタスクを混ぜない |
| ツールは変わる、フォーマットは残る | 全データをmarkdownで持つ |
| 自動化は補助輪 | 壊れても手動で同じことができる |

詳細は [設計/ARCHITECTURE.md](設計/ARCHITECTURE.md)

## カスタマイズ

| ファイル | 何を変えるか |
|---------|-------------|
| `CLAUDE.md` | 応答言語、ルール、完了宣言プロトコル |
| `設計/タグ定義.md` | プロジェクトに合わせたタグ体系 |
| `設計/テンプレート/` | 日報・月報のフォーマット |
| `.claude/settings.json` | 許可するBashコマンド |
| `.obsidian/app.json` | Obsidianの新規ファイル作成先、添付先 |

## 推奨Obsidianプラグイン

`/初期設定` 実行時にも案内されます。

- **Templater** — テンプレートから日報・月報を自動生成
- **Dataview** — ナレッジ・タスクの動的クエリ
- **Tasks** — チェックボックスタスク管理
- **Excalidraw** — 図・ホワイトボード
- **Git** — 自動バックアップ
- **Calendar** — カレンダービュー
