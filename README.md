# Obsidian + Claude Code テンプレート

Obsidian Vault と Claude Code の設定が一体化した汎用テンプレート。  
ダウンロードして Obsidian で開くだけで、ナレッジ管理 + AI自動化がすぐに使える。

## セットアップ

1. このリポジトリを clone または ZIP でダウンロード
2. Obsidian で「既存のフォルダをVaultとして開く」→ このフォルダを選択
3. Claude Code でこのフォルダを開く（`claude` コマンド実行）
4. 必要に応じて `CLAUDE.md` と `設計/タグ定義.md` をカスタマイズ

## フォルダ構成

```
├── inbox/                 ← 未整理素材の入口
│   ├── raw/               　 生データ
│   ├── temp/              　 一時ファイル（git管理外）
│   ├── まとめ済み/         　 処理済みインプット
│   └── ストック/           　 保留・あとで読む
│
├── ナレッジ/               ← 知識ベース（技術/ツール/リサーチ）
├── 議事録/                 ← 会議・面談記録
├── 日報/                   ← 日報・振り返り
├── タスク/詳細/            ← タスク個票
├── 制作/スライド/          ← 発表資料
├── ログ/チャット/          ← AIチャットログ
│
├── 添付/                   ← 画像等バイナリ（Obsidian添付先）
├── アーカイブ/             ← 処理済みファイル退避
├── Excalidraw/             ← 図・ホワイトボード
├── Clippings/              ← Webクリップ
│
├── 設計/                   ← Vault設計ドキュメント
│   ├── ARCHITECTURE.md     　 設計思想書
│   ├── タグ定義.md          　 タグ運用ルール
│   └── テンプレート/        　 日報・月報テンプレート
│
├── .claude/                ← Claude Code 自動化
│   ├── skills/ (16個)      　 汎用スキル
│   ├── agents/             　 エージェント定義
│   └── commands/           　 カスタムコマンド
│
├── .obsidian/              ← Obsidian 設定
├── CLAUDE.md               ← Claude Code プロジェクトルール
└── .gitignore              ← バイナリ/env/temp 除外
```

## 同梱スキル（16個）

| スキル | 説明 |
|--------|------|
| `/リサーチ` | Web検索で情報収集→ナレッジ保存 |
| `/リサーチ2スライド` | リサーチ→スライド一気通貫 |
| `/Webクロール` | 複数ページの再帰的クロール |
| `/Web取り込み` | URL→ナレッジ保存 |
| `/Webテスト` | Playwright でWeb動作テスト |
| `/defuddle` | Webページからクリーンなmarkdown抽出 |
| `/スプレッドシート` | Excel(.xlsx)作成・編集・分析 |
| `/ドキュメント作成` | Word(.docx)生成 |
| `/PDF操作` | PDF結合・分割・テキスト抽出 |
| `/GASコード作成` | Google Apps Script 生成 |
| `/スライド作成` | テーマ→構成案→PPTX生成 |
| `/深掘り` | アイデアを容赦なく質問して精緻化 |
| `/人間味` | AI臭い文章を人間らしく書き直す |
| `/obsidian-markdown` | Obsidian記法のmarkdown生成 |
| `/obsidian-bases` | Obsidian Bases（DB機能）作成 |
| `/json-canvas` | JSON Canvas でマインドマップ作成 |

## 設計思想

- **テキストは中に、バイナリは外に** — mdはVault内、音声・動画は外部
- **フォルダは役割、タグは性質** — フォルダ = 「何か」、タグ = 「何について」
- **1ファイル1目的** — 議事録とタスクを混ぜない
- **ツールは変わる、フォーマットは残る** — markdown で持つ

詳細は [設計/ARCHITECTURE.md](設計/ARCHITECTURE.md) を参照。

## カスタマイズ

| ファイル | 何を変えるか |
|---------|-------------|
| `CLAUDE.md` | 応答言語、ルール、完了宣言プロトコル |
| `設計/タグ定義.md` | プロジェクトに合わせたタグ体系 |
| `設計/テンプレート/` | 日報・月報のフォーマット |
| `.claude/settings.json` | 許可するBashコマンドやMCPサーバー |
| `.obsidian/app.json` | 新規ファイル作成先、添付先 |
