$ARGUMENTS で指定されたURLを起点に、ページ内のリンクを再帰的にたどりながら内容を取得・要約し、ナレッジとして保存してください。

引数（必須）: URL（1つ）
オプション:
- `深度:N` — リンクをたどる深さ（デフォルト: 2、最大: 3）
- `同一ドメイン` — 同じドメインのリンクのみたどる（デフォルト: ON）
- `全ドメイン` — 外部リンクもたどる
- `最大:N` — 取得する最大ページ数（デフォルト: 20、最大: 50）
- `カテゴリ:技術` — 保存カテゴリを指定（省略時は自動判断）
- `フィルタ:キーワード` — このキーワードに関連するリンクのみたどる

（例: `/Webクロール https://docs.example.com/guide 深度:3 最大:30`）
（例: `/Webクロール https://example.com/blog 全ドメイン フィルタ:AI`）

---

## ステップ0：前提情報を読み込む＆引数を解析

プロジェクト内にプロフィール情報やナレッジ一覧のREADMEがあれば読み込む。

引数から以下を解析：
- **起点URL**: 必須
- **深度**: `深度:N` から取得（デフォルト2）
- **ドメイン制限**: `全ドメイン` 指定がなければ同一ドメインのみ
- **最大ページ数**: `最大:N` から取得（デフォルト20）
- **フィルタ**: `フィルタ:キーワード` があれば関連リンクのみ対象
- **カテゴリ**: `カテゴリ:X` があればそれを使用

---

## ステップ1：起点ページを取得・分析

1. **WebFetch** で起点URLの内容を取得する
2. 取得できなかった場合はエラーを報告して終了
3. 内容を要約する（後でまとめに使う）
4. ページ内のリンクをすべて抽出する
5. 以下の基準でリンクをフィルタリングする：

**除外するリンク：**
- アンカーリンク（`#` のみ）
- JavaScript リンク（`javascript:`）
- メール・電話リンク（`mailto:`, `tel:`）
- 画像・動画・PDFなどのメディアファイル直リンク（`.png`, `.jpg`, `.mp4`, `.pdf` 等）
- ナビゲーション系（ログイン、サインアップ、利用規約、プライバシーポリシー）
- ドメイン制限が有効な場合、外部ドメインのリンク

**優先するリンク（`フィルタ` 指定時）：**
- リンクテキストやURLにフィルタキーワードを含むもの
- 記事本文のコンテンツエリア内にあるリンク

6. フィルタ後のリンク一覧を **ユーザーに提示** する：

```
## クロール計画

**起点:** {URL}
**検出リンク数:** {全リンク数} → フィルタ後: {対象リンク数}
**深度:** {N}
**最大取得数:** {N}

### たどるリンク一覧（深度1）:
1. {URL} — {リンクテキスト}
2. {URL} — {リンクテキスト}
...

続行しますか？（リンクを除外/追加したい場合は指示してください）
```

7. **ユーザーの承認を待つ**（確認なしで大量取得しない）

---

## ステップ2：再帰的にページを取得

ユーザーが承認したら、以下の手順で再帰的にクロールする。

### クロールのルール
- **訪問済みURL管理**: 同じURLは2度取得しない
- **最大ページ数**: 上限に達したら停止
- **並列取得**: Agent（サブエージェント）を活用して3〜5ページ同時に取得
- **深度管理**: 現在の深度を追跡し、上限を超えたら停止

### 各ページの処理

各ページについて以下を実行：

1. **WebFetch** で内容を取得
2. 取得失敗したらスキップ（エラーをログに記録）
3. メインコンテンツを抽出・要約（3〜5行）
4. 次の深度でたどるべきリンクを抽出（深度上限未満の場合のみ）

### 並列実行パターン

```
深度1のリンクが10個ある場合:
  → Agent A: リンク1〜3を取得・要約
  → Agent B: リンク4〜6を取得・要約
  → Agent C: リンク7〜10を取得・要約
  → 全エージェント完了を待つ
  → 結果を統合

深度2のリンク（深度1から検出されたもの）:
  → 同様に並列で取得
  → 最大ページ数の残りを各エージェントに配分
```

**サブエージェントへの指示テンプレート:**

```
以下のURLリストのページをWebFetchで取得し、各ページについて以下を返してください：
- URL
- ページタイトル
- 要約（3〜5行）
- 抽出したリンク一覧（{ドメイン制限の条件}）
- 取得成否

URLリスト:
1. {url}
2. {url}
...
```

### 進捗報告

5ページ取得ごと、または各深度の完了時にユーザーに進捗を報告：

```
進捗: {取得済み}/{最大} ページ（深度 {現在}/{最大}）
- 成功: {N}
- 失敗: {N}
- 残り: {N}
```

## ステップ3：内容を統合・構造化

すべてのページ取得が完了したら：

1. 全ページの内容を俯瞰し、**トピックごとにグルーピング** する
2. 重要度・関連度でソートする
3. 以下の構造でナレッジファイルを作成する

**カテゴリの判定（引数で指定されていない場合）：**

| カテゴリ | 保存先 | 判断基準 |
|----------|--------|----------|
| 技術・ツール | `ナレッジ/技術/` | プログラミング、ツールの使い方、設定方法 |
| AI・LLM | `ナレッジ/業務/AI・ツール/` | AI関連の技術、プロンプト |
| 時事・トレンド | `ナレッジ/時事/` | 業界ニュース、発表 |
| データ分析 | `ナレッジ/データ分析/` | 分析手法、SQL、統計 |
| 業務・ビジネス | `ナレッジ/業務/コンサル・リテール・オペ/` | リテールメディア、広告運用 |

### ナレッジファイルの形式

```markdown
---
type: ナレッジ
tags:
  - "#ナレッジ"
  - "#{カテゴリタグ}"
  - "#{トピックタグ1}"
  - "#{トピックタグ2}"
collected: {YYYY-MM-DD}
source_url: "{起点URL}"
crawl_depth: {実際にたどった深度}
pages_collected: {取得ページ数}
---

# {トピック名}

**取得日:** {YYYY-MM-DD}
**起点:** [{ページタイトル}]({起点URL})
**クロール範囲:** 深度{N}、{取得ページ数}ページ

---

## サマリー

{全体を俯瞰した5〜10行の要約。起点ページだけでなく、クロールで得た情報全体をカバーする}

---

## 詳細

### {トピック1}（{該当ページ数}ページから統合）

{複数ページの内容を統合・重複排除した要点}

- {要点1}
- {要点2}

**情報源:** [{ページタイトル}]({URL}), [{ページタイトル}]({URL})

### {トピック2}

...

---

## ページ一覧

| # | タイトル | URL | 深度 | 要約 |
|---|---------|-----|------|------|
| 1 | {タイトル} | {URL} | 0 | {一行要約} |
| 2 | {タイトル} | {URL} | 1 | {一行要約} |
| ... | | | | |

---

## 自分の業務への活用ポイント

- {具体的にどう活かせるか}
- {既存のプロジェクトとの関連}

---

## 関連ナレッジ

- [[{既存の関連ナレッジがあればリンク}]]
```

**ファイルが巨大になる場合（30ページ超）：**
- トピック別に複数ファイルに分割する
- インデックスファイル `{トピック名} - 目次.md` を作成し、各ファイルへのリンクをまとめる

---

## ステップ4：ナレッジREADMEを更新

ナレッジREADMEファイルに新規ファイルを追記：
- 該当カテゴリのセクションに `[[ファイル名]] — 一言説明` を追加
- 複数ファイルの場合はすべて追記

---

## ステップ5：完了報告

```
## Webクロール結果

**起点URL:** {URL}
**クロール範囲:** 深度{N}、{成功数}/{試行数} ページ取得
**保存先:** {ファイルパス}

### 全体サマリー（5行）
- {1}
- {2}
- {3}
- {4}
- {5}

### 取得したページ（主要なもの）
1. [{タイトル}]({URL}) — {一行要約}
2. ...

### 取得できなかったページ
- {URL} — {エラー理由}

### 業務への活用
- {具体的なアクション提案}
```

報告後、以下を確認：
- 「さらに深くたどりますか？（未訪問のリンクが{N}件あります）」
- 「特定のトピックについて追加リサーチしますか？」
