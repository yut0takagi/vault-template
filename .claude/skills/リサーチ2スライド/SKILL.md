$ARGUMENTS で指定されたトピックについてWebから網羅的に情報収集し、ナレッジ化した上で、LT用スライドまで一気に生成してください。

引数（必須）: トピック（枚数やトーン指定も可）
（例: `/リサーチ2スライド Claude Code MCPの使い方` `/リサーチ2スライド Vibe Coding 5分LT` `/リサーチ2スライド リテールメディア最新トレンド 10枚`）

---

## フェーズ1：リサーチ（情報収集＋ナレッジ化）

### 1-0. 前提読み込み

プロジェクト内にプロフィール情報やナレッジ一覧のREADMEがあれば読み込む。

### 1-1. Web検索で網羅的に情報収集（並列実行）

`$ARGUMENTS` のトピックについて、**Agentツールを並列で起動**して情報収集してください。

#### 並列化の方法

以下の **4つのサブエージェント** を **1つのメッセージで同時に起動** してください（subagent_type: "general-purpose"）：

| サブエージェント | 担当ソース | プロンプト概要 |
|-----------------|-----------|---------------|
| **Agent A** | Qiita + Zenn | WebSearch `"{トピック} site:qiita.com"` と `"{トピック} site:zenn.dev"` → 上位2-3件をWebFetchして要約 |
| **Agent B** | 海外技術ブログ | WebSearch `"{トピック英語} 2026"` → 上位2-3件をWebFetchして要約 |
| **Agent C** | 公式ドキュメント | WebSearch `"{トピック} official documentation"` → 公式ページをWebFetchして要約 |
| **Agent D** | プロジェクト内ナレッジ | Grep で `{トピック}` のキーワード検索 → 関連ファイルを読んで要約 |

各サブエージェントには以下を指示：
```
「{トピック}」について以下のソースから情報を収集してください。
検索して、信頼性の高いソース2-3件をWebFetchで取得し、以下の形式で返してください：
- ソース名とURL
- 要点（箇条書き5-10項目）
- 自分の業務への示唆
```

#### 結果の統合

全サブエージェントの完了後、メインエージェントが結果を統合してナレッジファイルを作成する。

- 各ソースから **合計5〜8件** の情報を収集
- 日本語と英語の両方で検索
- 公式ドキュメント・一次情報を優先

### 1-2. ナレッジファイルを作成

以下のフォーマットで保存（`/リサーチ` コマンドと同じ形式）：

**保存先:** `ナレッジ/技術/{トピック名}.md`（カテゴリは内容に応じて判断）

```markdown
---
tags: [ナレッジ, {カテゴリタグ}, {トピックタグ}, ...]
collected: {YYYY-MM-DD}
---

# {トピック名}

**調査日:** {YYYY-MM-DD}
**情報源:**
- [{ソース名}]({URL})
- ...

---

## サマリー
{3〜5行}

---

## 詳細
### {セクション1}
- ...

---

## 自分の業務への活用ポイント
- ...

---

## 関連ナレッジ
- [[...]]
```

### 1-3. ナレッジREADMEを更新

ナレッジREADMEファイルに追記してください。

## フェーズ2：LTスライド構成の設計

### 2-1. LTパラメータの決定

`$ARGUMENTS` からLTの設定を判断してください：

| パラメータ | デフォルト | 判断基準 |
|-----------|-----------|---------|
| **発表時間** | 5分 | 「5分LT」「10分」等の指定があればそれに従う |
| **枚数** | 8〜12枚 | 5分→8〜12枚、10分→15〜20枚。枚数指定があればそれに従う |
| **トーン** | カジュアル（LT向け） | 「真面目に」「提案用」等の指定があれば調整 |
| **対象者** | エンジニア・社内メンバー | 指定があればそれに従う |

### 2-2. LTスライドの構成テンプレート

以下の構成をベースに、トピックに合わせて調整してください：

**5分LT（8〜12枚）の場合：**

```
1. 表紙 — タイトル + 発表者名
2. 自己紹介（1枚） — 名前・所属・一言
3. 今日話すこと（1枚） — アジェンダ or 一言で掴む
4. 背景・課題（1〜2枚） — なぜこのトピックが熱いのか
5. 本題（3〜5枚） — リサーチで得た知見を凝縮
6. デモ or 具体例（1枚） — 実際に使ってみた / 事例
7. まとめ（1枚） — 3行で要点
8. 参考リンク（1枚） — 情報源URL一覧
```

**構成案をユーザーに提示して承認を得てください。**

```
【LTスライド構成案】（全XX枚・X分想定）

対象: {誰向け}
トーン: {カジュアル/フォーマル}

1. 表紙 — {タイトル案}
2. 自己紹介
3. ...

→ この構成で進めてよいですか？（タイトル・枚数・構成の変更があれば教えてください）
```

## フェーズ3：スライド生成

### 3-0. デザインシステムの読み込み

プロジェクト内にスライドアセット（slide_master.py等）があれば読み込む：

```python
import sys
# slide-assetsディレクトリへのパスを追加
from slide_master import (
    SlideDeck, add_text_box, add_rect, add_line,
    COLORS, FONT_SIZE, FONT_BODY_JP, FONT_BODY_EN,
    FONT_HEADING_JP, FONT_HEADING_EN, SLIDE_WIDTH, SLIDE_HEIGHT,
    CONTENT_AREA
)
from pptx.util import Emu, Pt
from pptx.enum.text import PP_ALIGN
```

### 3-1. LT向けデザインルール

通常のスライド作成ルールに加え、LT特有のルールを適用：

1. **1スライド1メッセージ（厳守）** — LTは流し読みされるので、1枚に詰め込まない
2. **フォントサイズ大きめ** — 本文18pt以上、見出し24pt以上を推奨
3. **箇条書きは3項目まで** — 5分LTで5行以上は読めない
4. **数字・データを大きく見せる** — `add_kpi_slide()` を積極活用
5. **コード例は最小限** — コードブロックは5行以内に収める
6. **「で、何が嬉しいの？」を常に入れる** — 技術紹介だけでなく、活用シーンを必ずセット

### 3-2. スライド生成

全スライドを **1つのPythonスクリプト** で生成。利用可能なメソッド：

| メソッド | 用途 | LTでの使いどころ |
|----------|------|-----------------|
| `add_title_slide(client, title, date)` | 表紙 | 1枚目。clientは空文字OK |
| `add_agenda_slide(items)` | アジェンダ | 「今日話すこと」 |
| `add_section_divider(num, title)` | セクション区切り | トピック切替 |
| `add_content_slide(header, sub, fn)` | 汎用 | 本題の各スライド |
| `add_numbered_cards_slide(h, s, cards)` | カード | ステップ説明・ポイント列挙 |
| `add_kpi_slide(h, s, kpis)` | 数値ハイライト | 統計・インパクトのある数字 |
| `add_comparison_slide(h, s, lt, li, rt, ri)` | 比較 | Before/After、ツール比較 |
| `add_table_slide(h, s, headers, rows)` | テーブル | 機能比較表 |
| `add_bullet_slide(h, s, bullets)` | 箇条書き | まとめ・参考リンク |
